TreeVisual

TreeVisual is an educational tool that provides an accessible web interface for students to explore and interact with graph visualizations of real-world data sets. A variety of modern graph layout algorithms (including animated, fully interactive physics-based simulations) allow users to discover patterns, clusters, and other properties of these data sets, which are drawn from a broad spectrum of fields, including geography, biology, and social networks. Users learn how to compare and evaluate different graph visualizations in side-by-side panels, which can be tweaked, deformed, and augmented by various interactions. Filtering and selection make even very dense graphs easy to visualize and understand.

Motivation

Graphs (or networks) are intuitive and familiar structures that use abstract node-link diagrams to represent relationships. Many different kinds of data, such as protein-protein interaction networks, brain connectivity networks, and social networks, have very natural graph representations, so graphs have found a wide variety of applications across many fields of study. Graph drawing is a fundamental topic in information visualization and thus is an essential part of a data visualization course.

Layouts

GraphVisual supports several force-directed layouts as well as random, circular, and spectral layouts. Force-directed layout algorithms aim to draw graphs that are aesthetically pleasing and easy to interpret. This is achieved by assigning attractive and repulsive forces between pairs of nodes and then using these forces to minimize an energy function.

GraphVisual allows users to change these forces (see Interactions) for the d3 force-directed layout, which is an implementation and adaptation of Tim Dwyer's [1] algorithm. Other force-directed algorithms include the fast multipole multilevel method (FM3) [2], ForceAtlas2 [3] which focuses on network spatialization, OpenOrd [4] which is designed for large graphs, and Yifan Hu's [5] algorithm which combines a multilevel approach, together with Barnes and Hut octree force calculation [6] and an adaptive cooling scheme for the general force model for efficient high-quality graph drawing.

The spectral graph layout uses the underlying information about the graph spectrum to place the nodes based on the eigenvectors of the graph's Laplacian matrix. We use the first two dominant eigenvectors as node positions for the spectral layout.

The circular layout arranges the nodes on a circle, and the random layout assigns node positions randomly.

In addition, GraphVisual supports two special layouts, geographical layout and bipartite layout, for drawing graphs with geographical information and bipartite graphs, respectively.

Data sets

GraphVisual comes with a set of example data sets from various fields. The data sets are drawn from the Network Data Repository with Interactive Graph Analytics and Visualization [7], Stanford Network Analysis Project [8], and the SuiteSparse Matrix Collection [9].

Interactions

GraphVisual provides essential interaction functions to allow users to compare different graph layouts in two side-by-side panels. There are options to adjust the appearance of the graph. For example, users can change the size of the nodes either in general or by node degree, or visualize edge weights via edge thickness.

Besides changing the appearance, users can filter the drawings by node degree, edge weight (for weighted graphs), or the length of the edges in the drawings. Additionally, users can select either nodes or node clusters in the graph to highlight them. The nodes are colored by their cluster labels obtained using the Louvain community detection algorithm [10].
