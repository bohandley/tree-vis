TreeVisual

TreeVisual is an educational tool that provides an accessible web interface for students to explore and interact with hierarchical data visualizations of real-world data sets. A variety of modern tree layout algorithms (including animations and fully interactive views) allow users to discover patterns, relationships, and other properties of these data sets, which are drawn from a broad spectrum of fields, including research, government, trade, and biology. Users learn how to compare and evaluate different hierarchical visualizations in side-by-side panels, which can be tweaked and augmented by various interactions. Filtering and selection make even very dense views easy to visualize and understand.

Motivation

Tree diagrams are used to respresent hierarchical data structures in a graphical form. They are given the name "tree" because classic representations resemble a tree, with the root at the base or beginning of the view and leaves at the end. Many different kinds of data can be respresented in a hierarchical format, such as govenerment structures, the tree of life, import and export data, and many more. Tree visualization is a fundamental topic in information visualization and thus is an essential part of a data visualization course.

Layouts

TreeVisual supports several layouts athat are intereactive and show various levels of each data structure. Some layouts are best suited for viewing leaves while others allow the user to dive into the structure from the root. Treemaps display a single layer of the tree structure at a time while radial trees can display the entire hierarchical structure.

The treemaps are divided into two layouts, a standard treemap displaying leaves and a zoomable treemap. The zoomable treemap allows the user to explore each level of the hierarchy, zooming in or out by clicking the parent of the displayed nodes. The standard treemap displays all the data's leaves in a variety of ways. The binary tiling strategy recursively partitions the specified nodes into an approximately-balanced binary tree, choosing horizontal partitioning for wide rectangles and vertical partitioning for tall rectangles. The slice divides the rectangular area specified by x0, y0, x1, y1 vertically according the value of each of the specified node’s children while the dice divides them horizontally. The slice and dice displays nodes alternately horizontally and vertically. (Need citations?)

The radial tree diagrams are closer to more traditional hierarchical views. Both diagrams display sets of children for each node from the root all the way to the leaves. The collapsible radial tree will expose the children of any node that is clicked.

Two additional views are used in TreeVisual, pack and sunburst. Pack diagrams use containment or nesting in circles to represent the hierarchiy. The size of each node's circle represents a cumulative size of the quantitative dimension of each leaf. The sunburst diagram depicts each paren/child relationship by concentric circles. The circle in the centre represents the root node, with the hierarchy moving outward from the center. By clicking any node the user may zoom into a view showing the clicked node and children of the node.

Data sets

TreeVisual comes with a set of example data sets from various fields. The data sets are drawn from various sources including publication data, government, trade and the tree of life.

Interactions

TreeVisual provides essential interaction functions to allow users to compare different graph layouts in two side-by-side panels. There are options to adjust the appearance of the graph. For example, users can change the size of the nodes either in general or by a value computed by accumulated leaves or a value of an accumulated size attribute on each leaf.

Besides changing the appearance, users can filter the drawings by each level, selecting any node between the root and leaves. Additionally, users can select leaves in the graph or in a list to highlight them. The nodes are colored by the children of the root node.
